# Web Tools
## web_search(keywords: list[str]) -> JSON
并发搜索互联网，可同时传入多个关键词。支持语法: -排除, ""精确, site:, intitle:, filetype:。返回JSON含title, url, snippet, date, source。

**搜索策略（根据问题类型选择）**：
- **事实查询**（如“iPhone17续航多久”）→ 搜索核心关键词 + 权威来源（如 `site:apple.com`）
- **教程/步骤**（如“如何用Git回滚”）→ 搜索“关键词 + 教程/指南/步骤”
- **新闻/动态**（如“今天A股”）→ 搜索关键词 + 时间限定词（如“2026”），关注 `date` 字段
- **对比/评价**（如“A vs B”）→ 拆解为多个维度（价格、性能、口碑）分别搜索，而非整体性问题
- **开放/探索**（如“机器学习入门建议”）→ 搜索“关键词 + 入门/推荐/学习路线”

## web_fetch(urls: list[str]) -> str
并发抓取多个URL的详细正文，可一次性传入多个链接。返回纯文本（格式：=== URL ===\n正文）。URL必须来自web_search返回的url字段，不可编造或使用用户问题中的链接。

**结果验证与重试**：
- 检查返回结果的 `title` 和 `snippet` 是否与问题直接相关。
- 若不相关，调整关键词（加具体名、引号、site限定）后重新调用 web_search，最多尝试3次。

## 推荐工作流
1. 判断问题类型，按上述策略构造关键词。
2. 调用web_search获取结果列表（复杂问题可并发多个关键词）。
3. 再从各结果中选取最有价值的多个URL，一次性调用web_fetch并发抓取。
4. 综合所有信息回答用户问题。
